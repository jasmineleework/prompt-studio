# Prompt Workbench - Mac 一键启动器

## 🚀 快速开始

### 最简单的使用方法
1. **双击 `start.command`** - 应用会自动启动并打开浏览器
2. 开始使用！

就是这么简单！🎉

## 📋 系统要求

- **macOS**: 10.15 (Catalina) 或更高版本
- **Node.js**: 16.0 或更高版本（如未安装，运行 `setup.command` 会引导您安装）
- **浏览器**: 任何现代浏览器（Chrome、Safari、Firefox、Edge）

## 📦 文件说明

```
prompt-workbench/
├── start.command      # 一键启动脚本（主要使用这个）
├── setup.command      # 首次安装/环境检查脚本
├── prompts/          # 您的 Prompt 文件存储位置
└── ...               # 其他应用文件
```

## 🛠 首次使用指南

### 选项 1：直接启动（推荐）
- 直接双击 `start.command`
- 脚本会自动检查环境并安装必要的依赖

### 选项 2：使用安装向导
- 双击 `setup.command` 运行安装向导
- 向导会：
  - 检查 Node.js 是否安装
  - 安装项目依赖
  - 创建配置文件

## 🎯 核心功能

- **项目管理**: 创建和管理多个 Prompt 项目
- **文件夹组织**: 使用文件夹分类管理项目
- **版本控制**: 保存每次编辑的版本，支持 diff 比较
- **本地存储**: 所有数据存储在本地，保护您的隐私
- **字符/Token 统计**: 实时显示 Prompt 字符数和预估 Token 数

## 🔧 常见问题

### Q: 双击 start.command 没有反应？
**A**: 可能是权限问题，在终端运行：
```bash
chmod +x start.command
./start.command
```

### Q: 提示"未找到 Node.js"？
**A**: 您需要先安装 Node.js：
1. 运行 `setup.command`，它会引导您安装
2. 或访问 https://nodejs.org/ 手动下载安装

### Q: 端口 3000 已被占用？
**A**: 启动脚本会自动检测并使用下一个可用端口（3001, 3002...）

### Q: 如何停止服务器？
**A**: 在终端窗口按 `Ctrl + C` 或直接关闭终端窗口

### Q: 数据存储在哪里？
**A**: 项目数据存储在浏览器的 IndexedDB 中，同时备份到 `prompts/` 文件夹

## 💡 提示和技巧

1. **保存版本**: 点击保存按钮，可以预览 diff 并添加版本描述
2. **拖拽**: 可以拖拽项目到文件夹中进行组织
3. **下载**: 使用下载按钮导出单个 Prompt 为 Markdown 文件
4. **重命名**: 双击项目或文件夹名称可以重命名
